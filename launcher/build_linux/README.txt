P2P Messenger - Інструкція з використання
=========================================

ВСТАНОВЛЕННЯ:

1. Розпакуйте завантажений ZIP-архів у зручне місце (наприклад, в домашню директорію)
2. Встановіть Docker і Docker Compose з репозиторіїв вашого дистрибутиву або з сайту:
   https://www.docker.com/products/docker-desktop/
3. Запустіть Docker і переконайтесь, що він працює (команда 'docker ps' повинна виконуватись без помилок)

ЗАПУСК (є два способи):

СПОСІБ 1 - Швидкий запуск:
1. Відкрийте термінал у розпакованій папці
2. Зробіть файл start.sh виконуваним: chmod +x start.sh
3. Запустіть: ./start.sh
4. Дочекайтесь запуску додатку у браузері

СПОСІБ 2 - Створення постійного додатку (рекомендовано):
1. Зробіть файл Create_Launcher.sh виконуваним: chmod +x Create_Launcher.sh
2. Запустіть: ./Create_Launcher.sh
3. Це створить файл 'P2P_Messenger.desktop' в тій же папці
4. Скопіюйте цей файл у ~/.local/share/applications/ для додавання в меню програм
   або просто запускайте його подвійним кліком з файлового менеджера

ВАЖЛИВО: При першому запуску Docker буде завантажувати необхідні компоненти,
         що може зайняти до кількох хвилин.

ЗБЕРЕЖЕННЯ ДАНИХ:
Усі ваші повідомлення, контакти та налаштування автоматично зберігаються
між запусками додатку. Вам не потрібно турбуватися про втрату даних -
вони зберігаються в Docker томах навіть після перезапуску комп'ютера.

УСУНЕННЯ ПРОБЛЕМ:
- Перезапустіть Docker, якщо виникають помилки
- Перший запуск займає більше часу через завантаження образів
- Переконайтесь, що порт 8000 не зайнятий іншими програмами
- Якщо браузер не відкрився автоматично, відкрийте http://localhost:8000
- Якщо виникають проблеми з базою даних, запустіть ./clean_docker.sh

ДОДАТКОВІ ФУНКЦІЇ:
- ./clean_docker.sh - очищення всіх даних і перезапуск з нуля
- ./access_db.sh - підключення до бази даних для перегляду і редагування

ЗУПИНКА:
1. Введіть 'exit' у консолі програми
2. Або закрийте термінал і виконайте в новому терміналі команду:
   cd "шлях до розпакованої папки" && docker-compose stop
